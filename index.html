《通信软件开发与应用》

一、简介

本次结课作业选择制作动态网页—图书管理系统。使用前端框架为Angular，通过对官网教程“英雄之旅”的学习，并通过自己查找资料，制作出此系统。

本系统主要的功能包括浏览图书、查找图书（支持模糊查找）、增加图书、修改图书和删除图书。

本系统数据是通过后台服务器Deployd和数据库MongoDB模拟远端服务器。

二、开发过程

1.模块规划

通过系统的功能需求，规划模块除过根模块之外分别是图书列表模块（book-list）、图书详情模块(book-detail)、图书修改模块(book-modify)、图书增加模块(book-add)、图书查找模块(book-find)共5个模块。图书删除功能内嵌到图书列表模块和图书详情模块之中。模块详细功能信息介绍如下：

  • 图书列表模块的主要作用是浏览图书，显示此系统所有的图书名称信息。通过点击列表图书名称跳转到图书详情模块，也可以在图书列表选择删除图书。
  • 图书详情模块的主要作用是显示图书信息，图书信息包括图书id、图书名称(book name)、作者(author)、出版社(publisher)和简介(introduction)。通过点击修改按钮跳转到图书修改模块，通过点击删除按钮删除图书。
  • 图书修改模块的主要作用是修改图书信息，支持修改的信息为图书名称、作者、出版社和简介。因为图书id是由数据库创建的唯一不可更改标识。
  • 图书增加模块的主要作用是增加图书，可以输入图书名称、作者、出版社和简介来增加图书信息。图书名称是不可以缺失的，其他三个信息可以缺失。增加成功后可以点击提示查看增加图书的详细信息并可以再修改。
  • 图书查找模块的主要作用是查找图书。通过图书名称来查找图书，支持模糊查找，点击查找结果可以跳转到图书详情模块查看图书详细信息。
  • 根模块主要放置了路由出口，来完成所有功能模块的路由展示。

import { Injectable } from '@angular/core';

        

        
import { Book } from './book';

        
import { Observable } from 'rxjs';

        
import { HttpClient } from '@angular/common/http';

        
​

        
@Injectable({

        
  providedIn: 'root'

        
})

        
export class GetbookService {

        
  private booksUrl = 'http://localhost:2403/bookdata';

        
​

        
  constructor(private httpClient: HttpClient) { }

        
​

        
  //获取所有图书

        
  getBooks(): Observable<Book[]> {

        
    return this.httpClient.get<Book[]>(this.booksUrl);

        
  }

        
  // 获取指定id的图书

        
  getBookById(id: string): Observable<Book> {

        
    return this.httpClient.get<Book>(`${this.booksUrl}/${id}`);

        
  }

        
  // 添加图书

        
  addBook(sbook: Book): Observable<Book> {

        
    return this.httpClient.post<Book>(this.booksUrl, sbook);

        
  }

        
  // 删除图书

        
  deleteBook(id: string): Observable<any> {

        
    return this.httpClient.delete(`${this.booksUrl}/${id}`);

        
  }

        
  // 更新图书

        
  updateBook(sbook: Book): Observable<Book> {

        
    return this.httpClient.put<Book>(this.booksUrl, sbook);

        
  }

        
}

  dbook: Book;

        
​

    private route: ActivatedRoute,

        
    private getBookService: GetbookService,

        
    private location: Location

        
  ) { }

        
​

  }

4.主要问题

本次开发遇到的主要问题在于

  1. 图书列表编号与增加图书的编号重复（已解决）

    起初图书类设置参数有id、no、name、author、publisher和introduction，id是数据库自动生成，而no用于图书代号，所以图书no不能重复，可以在增加图书函数中让输入的no和所有的图书no一一比较来解决此问题。也可以通过检查现有图书no自动生成与现有图书no不同的no默认放到输入框中，用户不能自己修改no。虽然这一问题可以通过以上两种方法解决，但是增加后的图书no想要在图书列表中从小到大排列起来却是比较难的问题。

  2. 删除图书后图书列表编号顺序难排列（已解决）

    因为删除的图书位置随机，所以很容易造成图书编号混乱，图书编号重复的问题，所以后来直接取消了no这一属性，图书列表直接顺序显示编号。

  3. 增加图书信息没有检查重复，理论不应该完全重复（未解决）

    增加的图书信息上传到数据库后会自动分配唯一id，因此要想检查图书信息不重复，只能在前端进行。而图书拥有的4个属性的值要与所有图书的对应属性一一比较即可解决此问题。但是由于知识受限，不确定这个是不是最好的解决方案，因此未能解决此问题，数据的问题最好到后端查重比较符合逻辑。

  4. 删除图书没有回收站功能（未解决）

    此功能就好比一个后悔药，删除过的图书信息可以找回，但是尝试过先把要删除的图书信息暂存到其他变量，再删除后，将暂存变量信息添加到数据库，但是结果不能成功，删除图书后暂存变量也会清空。目前没有找到更好的方法。

  5. 增加图书后不能自动跳转到图书详情模块展示刚添加的图书（已解决）

    起初想通过添加图书后获取全部图书信息，然后再拿出最后一个图书信息的id通过路由的方式跳转到图书详情模块，但是尝试后发现，后台服务器不能瞬间的将添加的图书信息更新出来。之后发现添加图书方法的返回值就是此图书信息，所以直接把返回值的id拿出来做路由。通过添加图书后，将添加成功消息做出路由的方式，点击添加成功消息即可跳转到此图书界面。

  6. 查找图书不能同时将图书id和name同时push出来（已解决）

    push()函数是将值添加到数组末尾，但是如果这个类包含多个属性（id和name）在网页无法显示出来，目前没有找到原因。但是通过使用两个数组，一个保存id，另一个保存name就解决了此问题。
